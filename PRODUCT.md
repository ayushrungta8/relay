# Product

## Register

product

## Platform

macos

## Users

Relay is for people who run several Codex tasks at once on a Mac and need to
stay focused in another application while that work continues. The primary job
is to understand what needs attention, what is still running, what changed, and
how much Codex capacity remains without repeatedly opening individual tasks.

## Product Purpose

Relay is a notch-native activity center and conversational controller for
Codex. It turns hidden task state, pending decisions, context consumption, and
account limits into a glanceable surface, then offers the smallest useful
action in place. Success means the user can supervise multiple Codex tasks from
the top of the screen and only open the full Codex app when deep inspection is
actually necessary.

## Positioning

Relay makes Codex background work legible and actionable without becoming a
second Codex client.

When Relay hands off new work, it always creates a new task that is visible in
Codex. Project work starts in the resolved existing project directory; general
questions and Mac or web actions start as normal projectless Codex chats. Relay
does not append new requests to an existing task or create hidden branches.

## Brand Personality

Calm, focused, and native. Relay should feel like a precise macOS system
surface: quiet while nothing needs attention, immediate when the user is
required, and fluid without becoming theatrical.

## Anti-references

Relay must not resemble a dense analytics dashboard, a wall of equally weighted
cards, a permanent floating widget, or a novelty Dynamic Island clone. It must
not duplicate full transcripts, diffs, settings, or project navigation that are
already better handled by Codex.

## Design Principles

1. Attention outranks activity, and activity outranks history.
2. The collapsed state communicates one useful fact or stays visually quiet.
3. Every visible metric must lead to a decision or explain current capacity.
4. Progressive disclosure keeps deep detail in Codex while preserving fast
   controls in Relay.
5. Motion explains the panel's origin, state, and continuity; it never delays
   work.

## Accessibility & Inclusion

All controls need meaningful VoiceOver labels and keyboard access. Status cannot
depend on color alone. Text uses semantic system styles. Large expansion motion
must reduce to a short crossfade when Reduce Motion is enabled, and the surface
must retain sufficient contrast over arbitrary desktop backgrounds.
